## 🚀 実行方法

### Supabase SQL Editorで実行

1. Supabaseダッシュボードにログイン
2. プロジェクトを選択
3. 左メニューから「SQL Editor」を選択
4. 「New query」をクリック
5. `create-demo-account.sql` の内容をコピー&ペースト
6. 「Run」ボタンをクリック

## 🔐 ログイン方法

1. アプリケーションのログインページにアクセス
2. 以下の情報を入力：
   - **企業コード**: `DEMO-2025`
   - **メールアドレス**: `demo@example.com`
   - **パスワード**: `DemoPassword123!`
3. 「ログイン」をクリック

## ⚠️ 重要な注意事項

### セキュリティ
- **本番環境では絶対にこのスクリプトを実行しないでください**
- デモ環境でのみ使用してください
- 本番環境で使用する場合は、必ずパスワードを変更してください

### データの削除
このスクリプトは、既存のデモアカウント（企業コード: `DEMO-2025`、メール: `demo@example.com`）を削除してから新しく作成します。

### 再実行
スクリプトは何度でも実行できます。既存のデモデータは削除され、新しいデータが作成されます。

## 🧪 デモアカウントでできること

デモアカウントにログインすると、以下の機能を試すことができます：

### 請求書管理
- 見積書の作成・編集・PDF出力
- 請求書の作成・編集・PDF出力
- メール送信（設定済みの場合）

### 取引先管理
- 3社のサンプル取引先を参照
- 新しい取引先の追加
- 取引先情報の編集

### 売掛・買掛管理
- 入金管理
- 支払管理
- エージングレポート

### 会計機能
- 勘定科目の参照（初回アクセス時に自動インポート）
- 仕訳入力
- 試算表
- 貸借対照表・損益計算書

### 経費精算
- 経費申請
- 領収書アップロード
- 承認フロー

### 固定資産管理
- 固定資産登録
- 減価償却計算
- 減価償却スケジュール

## 🔄 デモデータのリセット

デモデータを初期状態に戻したい場合は、もう一度 `create-demo-account.sql` を実行してください。

## 💡 カスタマイズ

スクリプト内の以下の変数を変更することで、カスタマイズできます：

```sql
demo_email text := 'demo@example.com';
demo_password text := 'DemoPassword123!';
demo_company_code text := 'DEMO-2025';
```

また、サンプル取引先や品目のデータも自由に変更できます。

## 📞 サポート

問題が発生した場合は、以下を確認してください：

1. Supabaseのデータベーステーブルが正しく作成されているか
2. RLS（Row Level Security）ポリシーが正しく設定されているか
3. トリガーが正しく動作しているか

それでも解決しない場合は、エラーメッセージを確認してトラブルシューティングを行ってください。
